# AGENTS.md — конвенции для всех агентов Synth

Этот файл читают все агенты (maker, checker, оркестратор) перед работой. Держи его коротким и точным.

## Проект
Synth — RAG-система для кодовых баз + VCS-автоматизации. Стек: **.NET** (бэкенд) + **Vue 3 + Vite** (фронтенд), **Qdrant** (вектора), **MongoDB** (конфиг). Код приложения живёт в `src/`.

## Общие правила
- **Одна задача — один worktree — одна ветка** `fix/<task-id>`. Не трогай файлы вне задачи.
- **Язык:** код и коммиты — на английском; финальные отчёты для человека — на русском.
- **Коммиты:** conventional commits (`feat:`, `fix:`, `chore:`, `test:`, `refactor:`), в теле ссылка на задачу `Task: <task-id>`.
- **Тесты обязательны.** Новый код без теста не считается готовым.
- **Не выдумывай правила приёмки** — они заданы в файле задачи и проверяются `scripts/validate.sh`.

## Границы ролей
- **maker** пишет код и тесты, гоняет сборку/тесты локально, но **не** создаёт PR, **не** запускает валидатор, **не** редактирует `loop/state/*`.
- **checker** **не** пишет и не правит код. Только запускает валидатор и возвращает вердикт `PASS`/`FAIL` дословно по его выводу.
- **оркестратор** (skill `loop`) управляет worktree, коммитами, состоянием — но не решает за валидатор, прошла задача или нет.

## Команды проекта (когда появится код в `src/`)
- Сборка: `dotnet build` / `npm --prefix src/client run build`
- Тесты: `dotnet test` / `npm --prefix src/client test`
- Ничего из этого пока нет — на этапе скаффолдинга валидатор проверяет только критерий приёмки.
