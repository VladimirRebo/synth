# Synth

> Личная RAG-система для кодовых баз + автоматизации вокруг VCS. Стек: **.NET + Vue**, векторное хранилище **Qdrant**, конфиг/реестр/граф-вызовов/логи — локально (файл + SQLite, `~/.synth/`).

Репозиторий начинался **не с приложения, а с agent loop** — автономного цикла, который сам берёт задачи, реализует их в изолированном worktree, независимо проверяет результат и записывает прогресс. Само приложение Synth разрабатывалось *внутри* этого цикла (и продолжает — вперемешку с прямой работой в сессии) и сегодня это полноценный рабочий инструмент, не заготовка.

Паттерн — [Loop Engineering](docs/loop-engineering.md) (Addy Osmani, 2026): ты проектируешь цикл, а не промптишь агента вручную.

## Что делает Synth

- **Индексация и семантический поиск** по кодовой базе — 5 чанкеров (C# через Roslyn AST, Python/Go/TypeScript+Vue через regex, Markdown/YAML/JSON для доков), эмбеддинги через Ollama/OpenAI, векторный поиск в Qdrant с ранжированием по типу чанка и ключевым словам.
- **MCP-сервер** (HTTP + stdio) — `search_code`, `get_symbol`, `get_file`, `find_callers`/`find_callees`, `index_code`, `list_collections`, `health_check` — так Synth сам становится инструментом для AI-агентов (Claude Code и других).
- **Структурный граф вызовов** — кто кого вызывает, извлекается синтаксически при индексации (не полноценная семантика, а эвристика по именам с языковой изоляцией — см. `IndexingPipeline.ResolveEdges`).
- **VCS-интеграция** — индексация по URL (GitHub/GitLab), периодический poll на новые коммиты, файловый watcher для локально индексируемых директорий (авто-переиндексация на изменения).
- **Multi-collection** — сколько угодно репозиториев/локальных директорий одновременно, поиск по одной коллекции или сразу по всем.
- **Vue-клиент** — индексация, поиск (с историей, фильтрами, deep-link в редактор), просмотр чанков файла, логи, настройки — то же самое, что MCP-инструменты, но для человека в браузере.

## Слой agent loop

```
.claude/
  agents/maker.md        # субагент-исполнитель (пишет код, границы жёсткие)
  agents/checker.md      # субагент-проверяющий (только вердикт по валидатору)
  skills/loop/SKILL.md   # логика оркестратора (как гонять один цикл)
  commands/loop.md       # /loop — запуск цикла из сессии Claude Code
  settings.json          # разрешения и хуки harness'а
scripts/
  loop.sh                # оркестратор: discover → isolate → make → verify → act
  validate.sh            # детерминированный валидатор (единственный источник правды о правилах)
  new-task.sh            # хелпер: создать файл задачи
loop/
  state/loop-state.md    # внешняя память цикла (Completed / Failures / In Progress)
  tasks/                 # задачи-контракты (по одной на файл)
docs/
  loop-engineering.md    # паттерн, адаптированный под Synth
src/                     # .NET-решение (Domain/Application/Infrastructure/Api + чанкеры) + Vue-клиент
```

## Быстрый старт

```bash
# 1. Посмотреть/создать задачу-контракт
./scripts/new-task.sh "Добавить эндпоинт /health"

# 2. Прогнать один цикл над первой задачей в очереди (демо-режим, без сети)
DEMO_MODE=true ./scripts/loop.sh

# 3. Боевой прогон (реально зовёт `claude`, правит код, коммитит в ветку)
./scripts/loop.sh loop/tasks/<task>.md
```

Или из сессии Claude Code: команда `/loop`.

Есть `Makefile` с типовыми командами (сборка, тесты, Aspire, клиент, обёртки над `scripts/*.sh`) — см. `make help`.

## Ключевые принципы цикла

- **Maker и checker никогда не видят работу друг друга** — проверка независима.
- **Критерий успеха — контракт, а не пожелание** (измеримая цель, проверяемые команды, границы, лимиты).
- **Валидатор (`scripts/validate.sh`) — единственный источник правды о правилах.** LLM-проверка не заменяет `dotnet test`/`build`.
- **Автономию наращивать по уровням** (см. [issues с меткой `roadmap`](https://github.com/VladimirRebo/synth/issues?q=is%3Aissue+label%3Aroadmap)) — не прыгать сразу на авто-merge.
